# 풀스택 개발자 가이드: 구현부터 운영까지 한 번에

## 목표
- 개발/배포/모니터링/복구까지 일관된 파이프라인 구축

## 권장 아키텍처
1. Starter(0원): 정적 사이트 + 플랫폼 기본 도메인
2. Growth: Next.js SSR + 최소 API
3. Full: Next.js + Auth + DB + Storage

## DevOps 기본 흐름
- 브랜치 전략: `main` 보호 + PR 기반
- CI: lint → test → build
- CD: main 머지 시 자동 배포
- 릴리즈: 롤백 가능한 배포 단위 유지

## Docker 도입 기준(초보자 보호 규칙)
- 정적 사이트만 운영하면 Docker 생략
- 서버 런타임이 필요할 때만 도입
- 로컬 테스트 포트는 `127.0.0.1` 바인딩 우선
- 비밀값은 이미지 빌드 단계에 넣지 않고 런타임 주입
- 데이터 영속성 필요 시 volume 필수

## 운영 문서 필수
- `.env.example`
- 도메인/DNS 설정 문서
- 장애 대응 런북
- 백업/복구 절차

- [ ] 신규 PC에서 30분 내 재현 가능
