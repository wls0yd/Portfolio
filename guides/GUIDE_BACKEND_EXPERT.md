# Backend 전문가 가이드: 홈페이지 API/문의 기능 안전 설계

## 목표
- 최소 기능 백엔드를 안정적이고 안전하게 운영

## 권장 원칙
- 0원/초기 단계에서는 백엔드 없이 정적 + 외부 폼 서비스부터 시작
- 서버는 필요한 만큼만(문의/관리/간단 API)
- 입력 검증 + 레이트리밋은 기본
- 비밀값은 환경변수/Secret Manager로 분리

## 기본 엔드포인트 예시
- `POST /api/contact`
- `GET /api/health`
- (선택) `POST /api/admin/*`

## 보안 체크리스트
- [ ] 요청 입력 검증(zod/joi)
- [ ] 레이트리밋 적용(문의/로그인)
- [ ] CAPTCHA 또는 bot 방지
- [ ] CORS 최소 허용
- [ ] 에러 추적(Sentry)

## 운영 체크리스트
- [ ] 4xx/5xx 로그 분리
- [ ] 요청 ID 기반 추적
- [ ] 장애 시 임시 차단/우회 절차 문서화
